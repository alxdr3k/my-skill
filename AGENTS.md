# my-skill 작업 규칙

## 절대 규칙

**사용자가 명시적으로 지시하기 전까지 `deploy.sh`를 절대 실행하지 않는다.**

커맨드 수정, 스크립트 수정, 구조 변경 등 어떤 작업을 완료하더라도 배포는 하지 않는다.
사용자가 "배포해", "deploy해" 등의 명령을 내릴 때만 실행한다.

## 작업 원칙

이 repo는 여러 프로젝트에 배포되는 agent command/skill/script의 원본이다. 작은 문구 변경도 다른 repo의 루프 동작을 바꿀 수 있으므로, 변경 전에는 의도를 명확히 하고 변경 후에는 실제 배포 산출물까지 상상해서 검토한다.

- **가정과 모호함 공개**: 애매한 요청은 조용히 한 해석으로 밀고 가지 않는다. 합리적 가정을 둔 경우에는 그 가정을 밝히고, 위험한 모호함이면 먼저 질문한다.
- **단순한 해결**: 요청 범위를 해결하는 최소 변경을 우선한다. 한 번만 쓰는 추상화, 미래 확장용 옵션, 불필요한 설정화는 추가하지 않는다.
- **좁은 diff**: 사용자의 요청과 직접 연결되는 줄만 바꾼다. 주변 문서/스크립트 정리나 포맷 변경은 필요한 경우에만 하고, 발견한 별도 문제는 임의 수정 대신 보고한다.
- **검증 가능한 목표**: 구현 지시를 성공 조건으로 바꿔 생각한다. 스크립트는 shell 문법과 대표 입력을 확인하고, command/skill은 agent가 실제로 그 지시를 어떻게 실행할지 예외 케이스까지 읽는다.
- **생성물 관점**: `commands/`, `codex/skills/`, `.agents/scripts/`는 서로 다른 agent 표면이다. 한쪽만 고치면 되는지, 같은 계약을 다른 표면에도 반영해야 하는지 확인한다.
- **남은 리스크 기록**: 해결하지 않은 위험, deferred/blocked 후보, 사용자 결정이 필요한 항목은 숨기지 말고 brief나 최종 보고에 남긴다.
